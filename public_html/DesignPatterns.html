Design Patterns in Calculator

This page contains a walk-through of the calculator program in python and a study on how design pattern principles are exhibited in this program


Overview of Design Patterns

To write readable and scalable code, we need to adhere to specific design patterns when writing code. In this calculator program, we employ several design pattern principles. The use of patters speeds up the development process by providing efficient code which assess problems before they even occur. Moreover, these patterns allow developers from different teams to communicate effectively as they share the same protocols.

Creational design patters are effective since they help delegate tasks to specific classes which perform one operation each. Class-Creation patterns use inheritance effectively (similar to our calculator implementation) and Object-Creation patters are centered around delegation of objects for different tasks.

The Factory Method

This is a creational design pattern which enables users to create a family structure of objects which share similar properties. Think of a chocolate factory. In a chocolate factory, several products such as biscuits, chocolate bars, wafers, etc. can all be produced. In a way, they all inherit from the chocolate factory class because they all share the same base ingredient, Chocolate. In our calculator, the Calculation class contains our concrete factory method

Here, from lines 8-11, we define our factory method which creates a Calculation object each time it is called by our Addition, Subtraction, Multiplication, and Division classes. The class method variable is set to the tuple of values that are passed into the factory method.

The Facade

This pattern provides an interface to interact with complex sets of classes within your program. It enables users to access the inner workings of your program while hiding the complexity of the program. Sounds familiar? (Hint: It's Abstraction).

For our calculator, we create a python file called calculator.py which handles all of the complex operations such as initialization, keeping track of dependencies, method execution etc. Think of this class as the buttons on a calculator which we can access without being concerned about the inner workings of that same calculator. This practice is especially useful when we want to expand our program to perform more functions. In sort, the facade class connects the user to the calculator program with minimal interaction.
